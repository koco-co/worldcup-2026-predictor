# 多场串关模式（竞彩过关 + 综合混合金字塔卡）

**何时进入：** 用户一次给了**多场比赛**（如「分析周二这 4 场」「这几场给个串关」），或明确要**串关 / 过关 / N 串 1** 方案时，进入本模式。单场比赛不构成串关——竞彩串关是**跨场**组合，同一场的不同玩法不能互串。

串关与复式规则（详见 [betting-rules.md](betting-rules.md) 第五节）：多场各取一注组成一注，**赔率相乘、任一场不中整串作废**；每条腿可**复式多选**（如一场押多个比分 / 多个方向），总注数 = 各腿选择数之积；**组合过关（M串N）** 注数 = 各 N-元组合的注数之和；可串场次、过关方式以官方为准。

> **综合串关卡＝混合金字塔（保底层走方向类、中层/封顶层走比分）。** 顶部置顶卡固定 3 张、围绕同一批 **4 场**核心场次**嵌套**：
> - **保底层 · 方向类混合过关（让球胜平负 / 胜平负）· 4串2/3/4 动态【推荐 · 主仓】**：取 4 场最可读方向，每场押 **1–2 个方向（双重机会）**；过关方式按整体置信**动态**——**置信高 → 4串3 / 4串4**（要求命中更多场、赔率更高）、**没把握 → 4串2+3+4 全包**（即 4串11，**2 场中即有奖、最宽容**）。**命中率最高、M串N 有容错，作主仓 / 固定推荐。**
> - **中层 · 比分复式 3串1**（3 场、**每场固定 2 档 → 8 注**）—— 不变。
> - **封顶层 · 比分单式 4串1**（4 场各 1 档 → 1 注）—— 不变。
>
> **为什么保底层从「比分串」改「方向类」（2026-06-25 复盘，结构性改版）：** 连续 4 批（6.21/6.23/6.24/6.25）比分金字塔全灭，**方向常对（6.25 方向 4/6 对）但比分容错为零**，方向判对的价值被「精确比分」玩法吃光。改用**方向类（让球/胜平负）+ M串N 组合过关（容错：不必全中）+ 每场双重机会（没把握押两种、有把握押一种）**，把主仓命中率优先级拉满；比分的高赔潜力保留在中层/封顶层的小注里。**胜平负 / 让球从此既是单场玩法、也是综合卡保底层的主玩法。**

## 流程

1. **逐场分析**：对每场跑分析（可用精简版——实力 / 状态 / 伤停 / 盘口 / 战意），每场产出：① 供**保底层**用的**最有把握方向**（让球胜平负 / 胜平负）+ **方向可读性**（能否给一个高置信单选方向，还是要双重机会兜）；② 供**中层 / 封顶比分卡**用的**比分读法**（首选比分 + 按对手类型的复式覆盖形状 + **比分可读性**）；③ 供**单场卡**用的最优玩法方向。
2. **按可读性分层排序**：**保底层**按**方向可读性**排（方向越确定越该进、且能单选不必双重机会）；**中层 / 封顶**按**比分可读性**排（比分难猜的场——铁桶 / 首发不定 / 要 4 档以上才盖得住——不进比分层，只配封顶或放弃）。

> **组卡前必过两道闸门（不占步骤编号，呼应 SKILL.md「对手类型分级」与「战意与控分分级」）：**
> **① 批次相关性闸门**：⚠️ **硬前置——填「对手 A/B/C 类型」前，先确认该对手近 1–2 轮真实数据已联网取齐（失球数、是否刚逼平/零封过强队）；无数据就标「类型待核实」、按中性处理，即便看起来像铁桶也不据此判 B、不收缩本批。** 然后把各场已打好的「对手类型(A/B/C) + 战意型 + 置信度」横向聚合一次，问：「这批是否共享同一系统性冷门源？」常见共享源——多场对手都是 B 纪律铁桶（尤其**近一轮刚逼平/零封过同级或更强队**）、整批多场低战意/已出线、整批轮换季/一周双赛首发不确定同源。**只要 ≥2 场共享同一冷门源，本批即「高相关批次」**——各场命中**不能再当独立事件连乘**（同坑会一起翻车），按下文「高相关批次收缩规则」主动收缩。⚠️ **触发收缩的 A/B/C + 冷门源必须是「已联网核实」的事实**——**未联网、或纯凭假设把多场默认判 B 再据此收缩，是 GIGO 误触发，禁止**（降级模式示例：三场无数据默认判 B → 砍仓，属误触发）。**无数据核实类型时一律不收缩整批、回到默认 3 卡。** **比分层（中层/封顶）方向一错比分必错、最致命；保底层方向类有 M串N 容错、对相关性更耐受，但同坑批仍会一起翻车，照样要收。**
> **② 选场闸门（分层，未通过者不得进对应层）：**
> - **保底层（方向类）收「方向最可读」的 4 场**：某场方向也难定（真冷平 / coin-flip / 首发乱）时，用**双重机会**兜（押两种，如「胜或平」「平或负」），或把该场排除出保底 4 场。**有把握的场押单一方向（如「让平」），没把握的场押双重机会**——这正是动态过关（4串2/3/4）+ 双重机会的设计目的：用「容错」换「不必每场都精准」。**方向反了照样全错（6.25 南非看反、瑞士判平），选方向时：生死方/主场搏命的实然处境 > 对手口头求胜的应然。**
> - **中层 / 封顶（比分）仍按「比分可读性」收**：一场命中「对手 B 纪律铁桶 / 首发未确认且轮换高 / 合理比分要 4 档以上才盖得住 / 风险栏写真冷平」任一项，即**不进中层**，只配进封顶层单式（彩票级、愿亏）或放弃。**禁止「比分难猜却硬塞进中层假装稳」。**
>
> ⚠️ 这两道闸门专治「把同坑批当独立赌注 / 把比分难猜的场塞进比分层假装稳」。

3. **组综合方案：固定 3 卡混合金字塔是「默认上限模板」，按批次相关性可收缩**（常规配资**保底层重、封顶层轻**）。默认投入：**保底层 ~72 · 中层 32 · 封顶层 20（基数约 128 区间）**：
   - **低相关批次（闸门未触发）→ 用默认 3 卡满铺。**
   - **高相关批次（≥2 场同坑）→ 主动收缩、不准满铺**，按下文「高相关批次收缩规则」砍场次、降串数/过关方式、降总投入。
   - **保底层 · 方向类混合过关（让球胜平负 / 胜平负）· 4串2/3/4【推荐 · 主仓】**：取 4 场最可读方向，每场 **1–2 个方向（双重机会）**，过关方式按整批置信动态选：
     - **整批方向把握高 → 4串3 / 4串4**（少容错或要求全中、赔率高）；
     - **整批没那么有把握 → 4串2+3+4 全包**（=4串11，**2/3/4 场中都有奖、最稳**）。
     - **注数**由所选过关方式与各场方向数（1–2）按 [betting-rules.md](betting-rules.md) 第五节正推：单选基数 **4串2=6 注 · 4串3=4 注 · 4串4=1 注 · 全包(4串2+3+4)=11 注**；某场押双重机会（2 个方向）时，含该场的组合注数再按选择数相乘。**作主仓、默认推荐**（命中率最高）。
   - **中层 · 比分复式 3串1**（不变）：在保底 4 场里取**比分最可读的 3 场**，每场**固定取 2 比分**，**注数固定 = 2×2×2 = 8 注**（默认 8 注 × 2 倍 = 32 元）。补「正好中 3 场比分」的回报。
   - **封顶层 · 比分单式 4串1**（不变）：4 场各押 1 个首选比分博高倍率，**注数固定 = 1 注**（默认 1 注 × 10 倍 = 20 元），彩票级、愿亏小注。

4. **标推荐**：固定推荐**保底层（方向类 4串2/3/4）**——方向类命中率高于比分、且 M串N 有容错，是三层里最该重仓的一层。方案名后加「**推荐**」，一句话说明。
5. **算投入与本票最高奖金（每注固定 2 元，正向乘出，禁止「总额÷注数」反推）**：先定**注数**（保底层按过关方式 + 双重机会正推；中层 = 各场比分数之积 = 8；封顶 = 1）与**倍数**（正整数）→ **投入 = 2 × 注数 × 倍数**（须为 2×注数 的整数倍）→ **本票最高可能固定奖金 = 2 × 倍数 × ∏(命中档赔率)**，与注数无关。赔率取官方实时值，取不到标估计。
6. **配资 + 止损**：默认基数 **约 128 元**（用户惯常区间 100–150、典型打法 72+32+20≈128；或用户当次给的金额）。混合金字塔**保底层重、封顶层轻**，默认按 **72 / 32 / 20**（保底方向类 / 中层比分 / 封顶比分）分配——把大头放命中率最高、有容错的保底方向类层，封顶层只用小注。
   - 每卡实际投入须吸附到 **2 × 该卡注数** 的整数倍（倍数为正整数），按 [betting-rules.md](betting-rules.md) 第五节落地，**绝不出现非法单注**。
   - **高相关批次**：先执行下文「高相关批次收缩规则」再分配剩余仓位，不机械满仓。
   - ⚠️ **结构风险提示（已随改版更新）**：保底层改方向类后，**整体比旧「三层全比分」稳健**（保底命中率高、有 M串N 容错）；但**中层 / 封顶仍是比分类高波动玩法**（命中率：中层 3串1 比分约个位数 %、封顶 4串1＜0.1%）。**求稳者可进一步加大保底层、砍中层 / 封顶层。** 总投入 ≤ 愿亏额度，不追高、不回本式加注。

### 保底层方向选择（让球胜平负 / 胜平负 + 双重机会）

**按该场战意 + 对手类型给保底层选方向；拿不准就上双重机会，别硬给单选：**

| 该场判断 | 保底层方向取法 |
| ---- | ---- |
| 强队碾压 / A 漏勺 / **生死需大胜**，方向高置信 | **单选**：让球胜（让 1 / 让 2 视首选净胜）或主胜 |
| **已出线控分避强 / 双方已出线只争名次**，可能平 | **双重机会**：胜或平（1X）或 平或负（X2），把「平」兜进去 |
| **真冷平 / coin-flip / 首发乱**，方向也难定 | **双重机会**兜，或干脆排除出保底 4 场 |
| 领头羊求稳零封小胜 | **单选**让平（让 1 后平）或主胜，视净胜把握 |

> **方向类比比分容错高，但方向反了照样全错。** 6.25 教训：南非（生死 + 主场）被对手「口头求胜」带成看反、瑞士（需赢争第一）被平局惯性压成判平。**选方向时，生死方 / 主场搏命的实然处境，优先级高于对手媒体/主帅的口头战意；势均力敌或低战意场一律用双重机会兜，不强给单选。**

### 比分复式覆盖形状（用于**中层 / 封顶**比分层，按对手类型反推，别默认窄区间小胜）

**对手 A/B/C 分级见 SKILL.md「对手类型分级」**——比分复式覆盖哪几档，由该场对手类型决定向哪一端展开。**先定首选比分，再按下表向一端加档，覆盖档要跨净胜球数分散方差**（押 2:0+2:1+1:0 全挤在净胜 1–2 的同簇 = 没分散）：

| 对手类型 | 覆盖向哪端展开 | 示例（首选 2:0） |
| ---- | ---- | ---- |
| **A 漏勺/实力悬殊碾压** | 向上含**大胜尾部**(净胜 3、4 球)，并勾 **「胜其他」兜底**防出网格 | 2:0 / 3:0 / 4:0 ＋「胜其他」 |
| **C 开放对攻/博客胜** | 含**「对方也进一球」的 BTTS 比分**(2-1/3-2/2-2)，别只押零封小胜 | 2:1 / 3:1 / 2:0 |
| **B 铁桶/类型不确定** | 比分难猜，**不进中层**，只放封顶层单式或放弃 | 不进中层 |

方向判对 ≠ 比分判对：碾压场最常错在「赢太多」(押 2:0、实际 4:0；**6.25 摩洛哥 2:0 实 4:2**)，对攻/客胜场最常错在「对方也进了」(押 2:0、实际 2:1；**6.25 波黑 2:0 实 3:1**)。覆盖形状就是堵这两端。**别为「包住一切」无限加档**——盖不住就把该场踢出比分层（放保底层方向类，或封顶单式），不是堆档摊薄期望。

### 高相关批次收缩规则（「批次相关性闸门」判定高相关时强制执行）

当闸门判定本批高相关（**≥2 场经已联网核实共享同一冷门源**；未联网/未核实不在此列；**若任一场类型无法核实，整批按低相关处理、不触发收缩**），默认 3 卡满铺会让金字塔整批一起翻车。按以下动作收缩，越相关收得越狠（依据只用**已核实的**置信度与对手 A/B/C 类型，不写死球队、不写死金额）：

| 动作 | 规则 |
| ---- | ---- |
| **砍场次** | 同坑的多场进比分层（中层/封顶）**最多保留 1 场**；保底层方向类可保留更多（有容错），但同坑场仍尽量分散。 |
| **降串数 / 过关方式** | 比分层串数能少则少；**保底层优先保住（方向类有容错）**，必要时把保底层过关方式从 4串3/4 放宽到 **4串2+3+4 全包**（更耐错），或缩为 3 场（3串2）。 |
| **先砍封顶 + 中层** | **封顶层 4串1、中层 3串1 最先砍**——比分层场次越多、同坑批方向一错越易全灭。高相关时**封顶层可直接归零**、中层缩减或取消，仓位收回保底方向类层。 |
| **降总投入** | 高相关批次**总投入显著下调**，省下的仓位留到低相关批次，不在高相关日满仓。 |

收缩后须在 md 里**写明**：因本批高相关（注明共享源），已把结构从默认 3 卡收缩为何（砍掉哪些同坑场、保底层过关方式如何放宽）。

**A 类崩盘漏勺/C 类开放对攻批次不触发收缩**：多场强队碰的是会送空间的 A 漏勺/C 对攻，那是利好同向、不是同坑，按常规默认结构处理——避免把该重仓的顺风批误杀成该收缩的逆风批。

**纪律：** 串关命中概率 ≈ 各场命中率连乘，随场次指数下降；**比分类玩法尤甚（多串多错）**。本结构**保底层已改方向类（命中率高、容错好），但中层/封顶仍是比分、整体仍带进取层**，必须给风险提示与最大不确定性、避免绝对化表达（呼应 SKILL.md 原则 5）。

## 顶部置顶「综合混合金字塔卡」数据块

串关报告（`predictions/<日期>/串关-*.md`）写完后，在 md **末尾追加一段 `type:"combo"` 的 FB-DATA 数据块**，脚本会把它渲染成 `index.html` **顶部置顶大卡**（在所有单场卡之上），把**固定 3 张串**各画成一张**体彩竞彩票面**（过关方式 + 倍数 + 合计 → 各场「主队 Vs 客队」选项@赔率 → 单倍注数 → 本票最高可能固定奖金），推荐固定在**保底层**。

- **保底层 legs 用 `play` 字段**（方向类，如 `"play":"让平 @3.87"` 或双重机会 `"play":"胜或平 (胜@1.19 / 平@5.40)"`），过关方式 `way` 写 `"4串2"` / `"4串3"` / `"4串4"` / `"4串2+3+4"`。
- **中层 / 封顶 legs 用 `scores` 字段**（比分，如 `"scores":[{"s":"2:1","o":7.0}]`），`way` 写 `"3串1"` / `"4串1"`。
- 字段格式见 [report-data.md](report-data.md) 的「综合串关报告」。`build_html_report.py` 已支持 `play` 与 `scores` 混合腿（6.25 实票②混合卡即用 `play` 渲染成功），保底层改方向类无需改渲染器。**没有数据块时**该报告退化成普通文本卡。
