# 多场串关模式（竞彩过关 + 综合比分金字塔卡）

**何时进入：** 用户一次给了**多场比赛**（如「分析周二这 4 场」「这几场给个串关」），或明确要**串关 / 过关 / N 串 1** 方案时，进入本模式。单场比赛不构成串关——竞彩串关是**跨场**组合，同一场的不同玩法不能互串。

串关与复式规则（详见 [betting-rules.md](betting-rules.md) 第五节）：多场各取一注组成一注，**赔率相乘、任一场不中整串作废**；每条腿可**复式多选**（如一场押多个比分），总注数 = 各腿选择数之积；可串场次、过关方式以官方为准。

> **综合串关卡＝比分金字塔（3 张全是比分串）。** 顶部置顶卡固定 3 张、围绕同一批核心场次**嵌套**：**保底层 2串1**（2 场、**每场固定复式 3 档 → 9 注**）· **中层 3串1**（3 场、**每场固定 2 档 → 8 注**）· **封顶层 4串1**（4 场、**每场 1 档单式 → 1 注**）。**已去掉胜平负卡**——胜平负 / 让球仍是**单场分析**里要判的玩法（进各场单场卡），但**综合卡只走比分**。三层比分「层层包含」（封顶层比分 ⊆ 中层 ⊆ 保底层），实际打出某比分时各层票同时存活；越上层场次越少、命中率相对越高。

## 流程

1. **逐场分析**：对每场跑分析（可用精简版——实力 / 状态 / 伤停 / 盘口），每场产出两样东西：① 供**单场卡**用的最有把握玩法方向（让球 / 胜负 / 总进球等）；② 供**综合卡**用的**比分读法**——首选比分 + 按对手类型的**复式覆盖形状**（见下「比分复式覆盖形状」）+ **比分可读性**（高 / 中 / 低：合理比分能用几档盖住）。
2. **按比分可读性排序**：把各场按**比分可读性**从高到低排；比分难猜的场（铁桶 / 首发不定 / 要 4 档以上才盖得住）**不要硬塞**进低串数的层，宁可只放进封顶层单式博彩票。

> **组卡前必过两道闸门（不占步骤编号，呼应 SKILL.md「对手类型分级」）：**
> **① 批次相关性闸门**：⚠️ **硬前置——填「对手 A/B/C 类型」前,先确认该对手近 1–2 轮的真实数据已联网取齐(失球数、是否刚逼平/零封过强队);无数据就标「类型待核实」、按中性处理,即便看起来像铁桶也不据此判 B、不收缩本批。** 然后把各场已打好的「对手类型分级(A/B/C)+置信度」横向聚合一次，问：「这批场次是否共享同一系统性冷门源？」常见共享源——多场对手都是 B 纪律铁桶（尤其**近一轮刚逼平/零封过同级别或更强球队**）、整批多场低战意/已出线、整批轮换季/一周双赛首发不确定同源。**只要 ≥2 场共享同一冷门源，本批即「高相关批次」**——各场命中**不能再当独立事件连乘**（同坑会一起翻车），按下文「高相关批次收缩规则」主动收缩。**比分串尤其致命：方向一错比分必错，整层一起灭。** ⚠️ **触发收缩的 A/B/C + 冷门源必须是「已联网核实」的事实**（近一轮真实失球、是否刚逼平/零封过强队等）——**未联网、或纯凭假设把多场默认判 B 再据此收缩，是 GIGO 误触发，禁止**（降级模式示例：三场无数据默认判 B → 保底缩单场 + 砍仓，属误触发）。**无数据核实类型时一律不收缩整批、回到默认 3 卡**；个别场即便已联网仍类型不明，按下条「保底层选场闸门」把那一场挡在保底/中层之外即可，不必把整批一起缩。
> **② 保底层选场闸门（优先级高于可读性排序，未通过者不得进保底层 2串1）**：保底层是金字塔里命中率相对最高的一层、**且它的两场出现在三层票里（错一场三层全废）**，所以只收「**比分最可读**」的两场。一场命中以下任一项即**不进保底层**——对手是 B 纪律铁桶 / 首发未确认且轮换风险高 / 合理比分要 **4 档以上**才盖得住 / 该场风险栏写了真实客胜或冷平。不合格场的三条出路、**禁止「比分难猜却硬塞进保底层假装稳」**：① **优先**换另一场「比分更可读」的已核实场顶上保底层，**尽量保住 2串1**；② 只有在全批确实**凑不出第二条已核实的干净腿**时，才把保底层从 2串1 缩为单场比分复式（**单场保底是极少数例外、不是默认**，缩了须在 md 里写明为何凑不出第二条干净腿）；③ 这种场可以只放进**封顶层单式**（彩票级、愿亏），不让它拖垮三层共用的保底两场。
> ⚠️ 这两道闸门专治「把同坑批当独立赌注 / 把比分难猜的场塞进保底层假装稳」（2026-06-22 复盘教训的比分版迁移）。

3. **组综合方案：固定 3 卡比分金字塔是「默认上限模板」，按批次相关性可收缩**（常规配资**保底层重、封顶层轻**）。**三卡注数与默认投入钉死（低/中相关满铺时）：保底层 9 注（3×3）= 72 元 · 中层 8 注（2×2×2）= 32 元 · 封顶层 1 注 = 20 元；合计约 124（基数 128 区间）。比分档数不是「2–3 档弹性」——保底层每场恰好 3 档、中层每场恰好 2 档、封顶层每场 1 档**：
   - **低相关批次（闸门未触发）→ 用默认 3 卡满铺**，下面三层不变。
   - **高相关批次（≥2 场同坑）→ 主动收缩、不准满铺**，按下文「高相关批次收缩规则」砍场次、降串数、先砍封顶/中层、降总投入。
   - **保底层 · 比分复式 2串1【推荐】**：取**比分最可读、已过「保底层选场闸门」**的 2 场，每场按对手类型**固定复式 3 档比分**（如 2:1 / 3:1 / 2:0，跨净胜球数分散方差），**中一即中**；**注数固定 = 3×3 = 9 注**（默认基数下 9 注 × 4 倍 = 72 元）。**禁止只取 2 档凑成 4 注**——某场合理比分凑不满 3 档，说明它本就不该进保底层（见「保底层选场闸门」），应换场或退出、不是降档。串数最少、命中率相对最高，综合首推。
   - **中层 · 比分复式 3串1**：在保底两场基础上**嵌套加 1 场**（可读性次高的一场），每场**固定取 2 比分**（**含保底层那档、保持层层包含**），**注数固定 = 2×2×2 = 8 注**（默认 8 注 × 2 倍 = 32 元）。补「正好中 3 场」的回报。
   - **封顶层 · 比分单式 4串1**：四场各押 1 个比分（取各场首选）博高倍率，**注数固定 = 1 注**（默认 1 注 × 10 倍 = 20 元），彩票级、愿亏小注。比分难猜的场只配进这一层。

4. **标推荐**：固定推荐**保底层（2串1）**（串数最少、每场复式覆盖、命中率相对最高），方案名后加「**推荐**」，一句话说明。
5. **算投入与本票最高奖金（每注固定 2 元，正向乘出，禁止「总额÷注数」反推）**：先定**注数**（复式串 = 各场比分数之积；单式串 = 1）与**倍数**（正整数）→ **投入 = 2 × 注数 × 倍数**（须为 2×注数 的整数倍，如 9 注复式只能 18/36/54…，**不能 40 元**）→ **本票最高可能固定奖金 = 2 × 倍数 × ∏(各场所选比分里的最高赔率)**，与注数无关。赔率取官方实时值，取不到标估计。
6. **配资 + 止损**：默认基数 **约 128 元**（用户惯常区间 100–150、典型打法 72+32+20≈128；或用户当次给的金额）。比分金字塔**保底层重、封顶层轻**，默认按 **72 / 32 / 20**（保底/中层/封顶）分配——把大头放命中率相对最高的保底层，封顶层只用小注。
   - 每卡实际投入须吸附到 **2 × 该卡注数** 的整数倍（倍数为正整数），按 [betting-rules.md](betting-rules.md) 第五节落地，**绝不出现 4.44 元这类非法单注**。
   - **高相关批次**：先执行下文「高相关批次收缩规则」（砍场次/降串数/先砍封顶+中层/降总投入）再分配剩余仓位，不机械满仓。
   - ⚠️ **必须在报告里点明结构风险**：本结构**三层全是「比分」类高波动玩法**，比旧「胜平负打底」更激进；命中率：保底层 2串1 比分约 10%、中层 3串1 更低、封顶层 4串1 ＜0.1%。**求稳者应自行加大保底层、砍中层/封顶层，或退回单场让球/胜负玩法。** 总投入 ≤ 愿亏额度，不追高、不回本式加注。

### 比分复式覆盖形状（按对手类型反推，别默认窄区间小胜）

**对手 A/B/C 分级见 SKILL.md「对手类型分级」**——比分复式覆盖哪几档，由该场对手类型决定向哪一端展开。**先定首选比分，再按下表向一端加档，覆盖档要跨净胜球数分散方差**（押 2:0+2:1+1:0 全挤在净胜 1–2 的同簇 = 没分散）：

| 对手类型 | 覆盖向哪端展开 | 示例（首选 2:0） |
| ---- | ---- | ---- |
| **A 漏勺/实力悬殊碾压** | 向上含**大胜尾部**(净胜 3、4 球)，并勾 **「胜其他」兜底**防出网格 | 2:0 / 3:0 / 4:0 ＋「胜其他」 |
| **C 开放对攻/博客胜** | 含**「对方也进一球」的 BTTS 比分**(2-1/3-2/2-2)，别只押零封小胜 | 2:1 / 3:1 / 2:0 |
| **B 铁桶/类型不确定** | 比分难猜，**不进比分金字塔**（保底/中层都别放），只能放封顶层单式或放弃 | 不进保底/中层 |

方向判对 ≠ 比分判对：碾压场最常错在「赢太多」(押 2:0、实际 4:0)，对攻/客胜场最常错在「对方也进了」(押 2:0、实际 2:1)。覆盖形状就是堵这两端。**别为「包住一切」无限加档**(竞彩每注 2 元、注数 = 各腿选择数之积)——盖不住就按「保底层选场闸门」**放弃该场**，不是堆档摊薄期望。

### 高相关批次收缩规则（「批次相关性闸门」判定高相关时强制执行）

当闸门判定本批高相关（**≥2 场经已联网核实共享同一冷门源**；未联网/未核实不在此列，见上「① 批次相关性闸门」；**若任一场类型无法核实，整批按低相关处理、不触发收缩**），默认 3 卡满铺会让比分金字塔整批一起翻车。此时按以下动作收缩，越相关收得越狠（依据只用**已核实的**置信度可靠度与对手 A/B/C 类型，不写死球队、不写死金额）：

| 动作 | 规则 |
| ---- | ---- |
| **砍场次** | 同坑的多场**最多保留 1 场**进串关，不把两场共享冷门源的场放进同一张卡；其余降级或放弃。 |
| **降串数** | 串关张数能少则少；**但保底层优先保住 2串1**（取同坑外两条最可读的已核实腿），只有确实凑不出第二条干净腿时，才把保底层缩为**单场比分复式**（极少数例外）。 |
| **先砍封顶 + 中层** | **封顶层 4串1、中层 3串1 最先砍**——场次越多、同坑批方向一错越易全灭。高相关时**封顶层可直接归零**、中层缩减或取消，仓位收回保底层。 |
| **降总投入** | 高相关批次**总投入显著下调**，省下的仓位留到低相关批次，不在高相关日满仓。 |

收缩后须在 md 里**写明**：因本批高相关（注明共享源），已把结构从默认 3 卡收缩为何（被砍掉哪些同坑场）。

**A 类崩盘漏勺/C 类开放对攻批次不触发收缩**：多场强队碰的是会送空间的 A 漏勺/C 对攻，那是利好同向、不是同坑，按常规默认结构处理——避免把该重仓的顺风批误杀成该收缩的逆风批。

**纪律：** 串关命中概率 ≈ 各场命中率连乘，随场次指数下降；**比分类玩法尤甚（多串多错）**。本结构**三层全是比分、偏进取**，必须给风险提示与最大不确定性、避免绝对化表达（呼应 SKILL.md 原则 5）。

## 顶部置顶「综合比分金字塔卡」数据块

串关报告（`predictions/<日期>/串关-*.md`）写完后，在 md **末尾追加一段 `type:"combo"` 的 FB-DATA 数据块**，脚本会把它渲染成 `index.html` **顶部置顶大卡**（在所有单场卡之上），把**固定 3 张比分串**各画成一张**体彩竞彩票面**（过关方式 + 倍数 + 合计 → 各场「主队 Vs 客队」比分@赔率 → 单倍注数 → 本票最高可能固定奖金），推荐固定在**保底层**。字段格式见 [report-data.md](report-data.md) 的「综合串关报告」。**没有数据块时**该报告退化成普通文本卡。
