# HTML 图表数据块（FB-DATA）规范

`scripts/build_html_report.py` 把每个日期目录的 md 报告渲染成一个 Claude 风格 `index.html`。卡片**折叠态只显示标题 + 摘要 chip**；**展开后顶部显示图表**（胜平负概率条、预期进球、Top-N 比分、总进球分布、让球倾向、押注方案），下方是 Typora 式正文（左侧大纲 + 可折叠标题）。这些图表，以及顶部**置顶综合串关卡**，都来自每篇 md 里嵌的一段**机器可读数据块**。

写报告时，在 md 末尾追加一段 HTML 注释包裹的 JSON（注释不会在正文里显示）。**没有数据块也能跑**——脚本回退成纯文本卡，只是没有图表。所以图表是增强项，但**强烈建议带上**。

## 写在哪、什么格式

在 md 的**最末尾**追加：

```
<!--FB-DATA
{ ...JSON... }
-->
```

- 必须是**合法 JSON**（双引号、无尾逗号、无注释）。解析失败脚本会忽略它、回退纯文本卡。
- 数值用**数字**不带 `%`（写 `55` 不是 `"55%"`）。概率数组尽量加和 ≈ 100，脚本会按比例归一，不强制。
- 字段都可选，**有就画、没有就跳过**——拿不到赔率隐含概率就省略 `odds`，拿不到让球盘就省略 `hhad`。

## 单场报告（`type:"match"`，可省略 type）

```json
{
  "match": "荷兰 vs 瑞典",
  "league": "世界杯小组赛",
  "kickoff": "2026-06-21 01:00",
  "result": "主胜（荷兰）",
  "confidence": "中",
  "score": "1:0",
  "recommend": "让球主胜（2串1腿）",
  "odds_time": "截至 17:41",
  "wdl": { "home": "荷兰", "away": "瑞典", "qual": [55, 24, 21], "odds": [50, 26, 24] },
  "xg": [1.77, 0.87],
  "score_top": [
    { "s": "1-0", "p": 12.6, "pick": true },
    { "s": "2-0", "p": 11.3 },
    { "s": "1-1", "p": 10.8 }
  ],
  "ttg": [3, 12, 24, 28, 18, 9, 4, 2],
  "hhad": { "line": "主-1", "probs": [48, 27, 25] },
  "plans": [
    { "name": "稳健", "tag": "推荐", "play": "让球主胜·单关", "cover": "让胜", "odds": 1.85, "stake": 50, "max": 92.5, "note": "命中率优先" },
    { "name": "中庸", "play": "总进球·复式", "cover": "2,3 档", "odds": 1.40, "stake": 30, "max": 42, "note": "" },
    { "name": "搏大", "play": "比分·复式", "cover": "1-0,2-0,2-1", "odds": 8.5, "stake": 20, "max": 56.7, "note": "复式3注，博精确" }
  ]
}
```

字段含义：

- 顶层 `match/league/kickoff/result/confidence/score/recommend`：卡片标题与摘要 chip（与汇总表一致；脚本若没读到数据块，会从「最终结论汇总表」回退提取这些）。
- `wdl`：胜平负概率。`qual` = 分析师**定性估计** `[主, 平, 客]`；`odds` = **官方赔率去水隐含**（可省）；`home/away` 队名用于标注。脚本画**两行概率条**对比（定性 vs 市场）。
- `xg`：`[主, 客]` 预期进球。
- `score_top`：Top-N 比分，`p` 概率，`pick:true` 表示被押注覆盖（复式高亮）。
- `ttg`：总进球 `0,1,2,3,4,5,6,7+` 共 **8 档**概率（竞彩进球玩法，**不是大小球**）。
- `hhad`：让球后倾向，`line` 让球文案，`probs` = `[让胜, 让平, 让负]`。
- `plans`：单场押注方案；`cover` 覆盖的选择（复式写多个），`odds` 参考赔率，`stake` 投入，`max` = **理论最高奖金**（按 [betting-rules.md](betting-rules.md) 第五节算），`tag:"推荐"` 高亮。

## 综合串关报告（`type:"combo"`，写在 `串关-*.md` 里）

当天有多场、产出串关综合报告时（见 [parlay-mode.md](parlay-mode.md)），在该 md 末尾写 `type:"combo"` 的数据块。脚本识别后渲染成**顶部置顶大卡**，每个方案画成一张**体彩竞彩票面**——从「过关方式」那一行起：过关方式 + 倍数 + 合计 → 各场（主队 Vs 客队）比分@赔率 → 单倍注数 → 本票最高可能固定奖金。**综合卡固定 3 张「比分串关」、构成比分金字塔**（保底层 2串1 / 中层 3串1 / 封顶层 4串1），**已去掉胜平负卡**：

```json
{
  "type": "combo",
  "date": "2026-06-23",
  "stake_total": 124,
  "odds_time": "赔率取官方SP，部分为估值",
  "combos": [
    {
      "name": "保底层 · 比分复式 2串1", "way": "2×1", "mult": 4, "tag": "推荐",
      "legs": [
        { "no": 1, "id": "041", "home": "阿根廷", "away": "奥地利",
          "scores": [ {"s":"2:0","o":6.000}, {"s":"2:1","o":6.500}, {"s":"3:1","o":9.250} ] },
        { "no": 2, "id": "042", "home": "法国", "away": "伊拉克",
          "scores": [ {"s":"3:0","o":5.100}, {"s":"3:1","o":10.00}, {"s":"4:0","o":6.000} ] }
      ],
      "bets": 9, "stake": 72, "max": 740,
      "hit": "两场各命中所选比分之一", "risk": "三档里命中率最高；两场须全中"
    },
    {
      "name": "中层 · 比分复式 3串1", "way": "3×1", "mult": 2,
      "legs": [
        { "no": 1, "id": "041", "home": "阿根廷", "away": "奥地利",
          "scores": [ {"s":"2:0","o":6.000}, {"s":"2:1","o":6.500} ] },
        { "no": 2, "id": "042", "home": "法国", "away": "伊拉克",
          "scores": [ {"s":"3:1","o":10.00}, {"s":"3:0","o":5.100} ] },
        { "no": 3, "id": "043", "home": "挪威", "away": "塞内加尔",
          "scores": [ {"s":"2:1","o":6.750}, {"s":"2:0","o":7.000} ] }
      ],
      "bets": 8, "stake": 32, "max": 1820,
      "hit": "三场各命中所选比分之一", "risk": "比 2 串 1 多一场，命中率下降"
    },
    {
      "name": "封顶层 · 比分单式 4串1", "way": "4×1", "mult": 10,
      "legs": [
        { "no": 1, "id": "041", "home": "阿根廷", "away": "奥地利", "scores": [ {"s":"2:0","o":6.000} ] },
        { "no": 2, "id": "042", "home": "法国", "away": "伊拉克", "scores": [ {"s":"3:1","o":10.00} ] },
        { "no": 3, "id": "043", "home": "挪威", "away": "塞内加尔", "scores": [ {"s":"2:1","o":6.750} ] },
        { "no": 4, "id": "044", "home": "约旦", "away": "阿尔及利亚", "scores": [ {"s":"0:1","o":5.600} ] }
      ],
      "bets": 1, "stake": 20, "max": 45360,
      "hit": "四场比分全中（倍率极高）", "risk": "命中＜0.1%，愿亏彩票"
    }
  ]
}
```

字段含义（票面口径）：

- `stake_total` 总配资（未给金额时默认 **100**，给了就用给的；上例用真实票面 124）；`combos` 为**固定 3 张比分串关**、构成「比分金字塔」（见 [parlay-mode.md](parlay-mode.md)）：
  - **combo[0] 保底层 · 比分复式 2串1（固定 `tag:"推荐"`）**：2 场比分、每场可复式多选；串数最少、命中率相对最高。
  - **combo[1] 中层 · 比分复式 3串1**：在保底层基础上嵌套加一场，补「中 3 场」的回报。
  - **combo[2] 封顶层 · 比分单式 4串1**：四场各 1 个比分博高倍率，彩票级。
  - **三层比分应「层层包含」**（4串1 的比分 ⊆ 3串1 ⊆ 2串1），实际打出某比分时各层票同时存活。**已去掉胜平负卡**——胜平负仍是单场分析的玩法，但综合串关卡只走比分金字塔。
- 每个 combo（票面字段）：
  - `name` 卡片标题（金字塔标签：保底层 / 中层 / 封顶层）；`tag:"推荐"` **固定标在 combo[0]**。
  - `way` 过关方式（`"2×1"` / `"3×1"` / `"4×1"`）；`mult` 倍数（正整数）；`stake` 合计投入。
  - `legs[]` 各场：`no` 第几场、`id` 场次编号（如 `"041"`，可省）、`home`/`away` 主客队、`scores[]` 该场所选比分（`s` 比分、`o` 赔率；单式 1 个、复式多个）。
  - `bets` 注数（单式串 = 1；复式串 = **各场比分数之积**，如 3×3=9、2×2×2=8）；`stake` = 2 × `bets` × `mult`，须为 2×`bets` 的整数倍。
  - `max` = **本票最高可能固定奖金 = 2 × `mult` × ∏（各场所选比分里的最高赔率）**，与注数无关（见 [betting-rules.md](betting-rules.md) 第五节）。
  - `hit` 命中条件，`risk` 风险一句话（中层 / 封顶须注明命中率随场次下降、愿亏）。
- **显示**：脚本对赔率用 `fmtO` 按票面补零（<10 三位、≥10 两位，如 `6.000` / `10.00`），对金额用 `fmtM` 加千分位（如 `45,360.00`）——所以 JSON 里照常写数字即可。
- **向后兼容**：旧版 `legs[].match/play/sel/odds` 数据块仍能渲染（退化成腿列表）；新报告一律用上面的票面字段。

## 诚实纪律

- 概率是**分析师定性估计**（`qual`）或**赔率去水隐含**（`odds`），**不是模型计算、不是保证**——卡片会标注、页脚有免责。两者差异正是找价值 / 冷门的看点。
- 赔率 / 隐含概率拿不到时**省略对应字段，不要编**。
- `max`（理论最高奖金）是命中最优时的**上限、非保证**；须与正文「押注方案」表的数字保持一致。
