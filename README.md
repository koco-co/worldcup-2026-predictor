# ⚽ worldcup-2026-predictor

> 面向 **2026 FIFA 世界杯**的赛前比分预测 → 赛后复盘 → 策略量化优化**闭环**。
> 每场给出结构化深度预测(竞彩五大玩法),赛后对照真实赛果复盘,再用 Dixon-Coles 比分矩阵回测验证规则是真改进还是过拟合。

---

## ✨ 这个项目在做什么

这是一个围绕 2026 世界杯的足球比赛分析项目,核心是一条**可迭代的预测闭环**:

1. **赛前预测** — 每场一份结构化深度分析,落到中国体彩**竞彩**五大玩法(胜平负 / 让球胜平负 / 比分 / 总进球数 / 半全场,**无大小球**)的选注与串关。
2. **赛后复盘** — 对照实际比分逐场复盘,定位系统性偏差(如「判了开放对攻却没把总进球升够档」),沉淀成**可执行规则**。
3. **量化优化** — 用 Dixon-Coles 比分概率矩阵(ρ 低分修正)做跨批次回测,对照旧 λ(OLD)与新规则 λ(NEW),确认新规则是真改进还是只过拟合当批。

## ⚠️ 适用范围

**本项目只针对 2026 FIFA 世界杯**积累数据、复盘与优化策略。对其它联赛 / 杯赛**没有积累数据**,这里的预测口径与规则**未必适用**于其它赛事,请勿直接套用。

## 🚀 对阵全景平台

仓库根的 [`index.html`](index.html) 是一个**自包含**的赛事面板(对阵全景 / 每日预测 / 积分榜 / 数据中心)——数据已内联,**直接用浏览器打开即可**,无需服务器、可离线。改完真源数据后跑一次:

```bash
./init.sh        # = python3 scripts/build_worldcup_data.py
```

它把赛程 / 赛果(`tournament.json`)与押注台账(`betting-ledger.json`)同步进 `data/worldcup-data.js` 并内联进 `index.html`。

## 📁 目录结构

```
index.html                 对阵全景平台(站点入口 / GitHub Pages 根;自包含,数据已内联)
init.sh                    一键同步数据
data/worldcup-data.js      数据契约(DATA/LEDGER 自动生成;国旗/时区/报告链接手工维护)
scripts/build_worldcup_data.py  从真源生成数据并内联进 index.html
赛前预测/<YYYY-MM-DD>/      每场一个 .md 深度分析 + index.html 可视化卡片
赛后复盘/<YYYY-MM-DD>/      该批次赛后复盘(预测 vs 实际、命中、系统性偏差)
赛后复盘/回测/             跨批次回测:.md 回测报告 + dc 输入 .json
.claude/skills/football-match-analysis/   分析方法论 skill(开源)+ 数据真源
```

> 说明:预测由 `football-match-analysis` skill 驱动,**方法论随仓库公开**(`.claude/skills/`);数据真源(赛程、押注台账)亦在 skill 的 `assets/` 内。

## 🔬 方法与工作流

| 阶段 | 产出 | 关键动作 |
| ---- | ---- | ---- |
| 预测 | `赛前预测/<日期>/*.md` + `index.html` | 球队画像、对手类型分级(A 崩盘漏勺 / B 纪律铁桶 / C 开放对攻)、首发确认、竞彩选注、比分金字塔串关 |
| 复盘 | `赛后复盘/<日期>/*.md` | 预测 vs 实际、逐场命中、实体票结算、系统性偏差归因 |
| 回测 | `赛后复盘/回测/*.md` + `*.json` | 把新规则跨批次回放,Dixon-Coles 矩阵量化对照 OLD vs NEW |
